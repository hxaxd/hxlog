## 项目经历

### 基于 PI Agent 的 Coding Agent | 独立开发 | 2026.1 - 至今

**项目描述**：项目聚焦上下文工程、工具调用、可审计执行、长期记忆与评测闭环，使 Agent 能够在已有代码仓库中完成需求理解、代码修改、测试执行、错误修复与变更总结。

**主要工作**：

* 参考 Pi 的极简 Agent 设计，将系统拆分为 **Agent Loop、上下文管理、工具系统、记忆系统、任务状态机、评测模块** 等核心组件，避免过度依赖黑盒框架，保证每次模型输入、工具调用与输出结果可检查、可复现。
* 设计 Coding Agent 的上下文组织策略，支持基于仓库结构的渐进式披露：优先读取 README、目录索引、`AGENT.md`、测试文件与调用链，再按任务需要逐步展开实现细节，降低一次性塞入无关代码导致的上下文污染。
* 实现面向代码仓库的工具调用层，封装文件读取、代码搜索、精确编辑、Shell 执行、测试运行、Git diff 查看等基础工具，并为每类工具设置参数约束、执行日志与错误回传机制。
* 设计任务级记忆与项目级记忆机制，记录仓库约定、常用命令、历史失败原因、测试入口、代码风格与模块边界，使 Agent 在多轮任务中能够复用经验，而不是每次重新探索仓库。
* 引入 Spec-Driven 与 TDD 流程：先将自然语言需求转化为可检查的任务规格，再生成测试或验证脚本，随后进行最小代码修改，并通过自动化测试与 diff 检查确认改动范围。
* 针对 Agent 常见的“误改无关文件、跳过测试、幻觉 API、重复尝试失败方案”等问题，设计执行前计划、执行中检查点、执行后变更总结与人工确认机制，提升复杂任务中的可控性。
* 构建 Agent 效果评测集，覆盖 Bug Fix、功能新增、重构、文档补全、测试补全等典型编码任务，以任务完成率、测试通过率、修改文件数、无关 diff 比例、平均迭代轮次等指标评估 Agent 能力变化。
* 支持将单次任务过程导出为结构化 trace，包括用户目标、上下文读取路径、工具调用序列、失败原因、代码 diff 与最终验证结果，用于复盘 Agent 决策质量与优化 prompt / 工具设计。
* 在本地工程实践中验证该 Agent 可完成中小规模仓库的功能补全、测试修复、接口迁移与文档生成任务，并可通过项目记忆持续降低后续任务的上下文探索成本。
