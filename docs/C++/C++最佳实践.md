## 使用头文件优化编译时检查

- 源文件应包含其配对的头文件有助于将链接时错误转化为编译时错误 (函数返回值不一致)
- 头文件使用了另一个头文件的内容, 应该直接包含该头文件, 而不是依赖于使用者的包含顺序, 所以我们的包含顺序应该反过来
    - 配对头文件
    - 同项目头文件
    - 第三方库头文件
    - 标准库头文件
- 避免依赖传递包含

## 避免使用无符号整型

- 无符号整型在算术运算中可能导致意外的结果
- 除外情况
    - 位运算
    - 需要明确表示非负值的场景
    - 需要明确的环绕行为的场景

## 小问题

- 不要用父类指针遍历子类对象的数组
    - 父类指针算术运算的步长是父类大小, 可能无法正确遍历子类对象
    - `delete[]` 也会有类似问题
