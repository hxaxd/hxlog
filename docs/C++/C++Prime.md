## IO 流

- 标准流对象
    - `std::cin` 是 `std::istream` 对象
    - `std::cout`, `std::cerr`, `std::clog` 是 `std::ostream` 对象
    - `std::cerr` 常用于错误信息, 默认更偏向立即输出
    - `std::clog` 常用于日志信息, 默认更偏向缓冲输出
- `std::endl` 会写入换行并刷新缓冲区
    - 普通换行优先使用 `'\n'`
    - 只有确实需要立刻刷新时再使用 `std::endl`
- 流对象可以用于条件判断
    - `while (std::cin >> value)` 是典型写法
    - 读取失败, EOF 或格式错误会使流进入错误状态
    - 流进入错误状态后, 后续输入不会继续正常读取, 需要先恢复状态
- 常见流状态
    - `eofbit`: 到达输入结尾
    - `failbit`: 格式不匹配或读取失败
    - `badbit`: 底层 IO 错误
    - `clear()` 可以重置状态
    - `ignore()` 可以丢弃残留输入
- `operator>>` 和 `std::getline` 的差异
    - `operator>>` 跳过前导空白, 遇到后续空白停止
    - `std::getline(is, s)` 读取到换行符为止, 并丢弃换行符
    - 混用二者时, 格式化输入留下的换行符可能让下一次 `getline` 读到空行
    - 常见处理方式是 `std::getline(std::cin >> std::ws, line)`

## 字符串与 C 接口

- `std::string::size_type` 是无符号类型
    - 与负数或有符号整数混用时容易出现隐式转换问题
    - 需要有符号长度时优先考虑 `std::ssize`
- `std::string::c_str()` 返回指向内部字符数组的指针
    - 指针只在字符串对象仍然存在且未发生相关修改时有效
    - 如果 C API 会保存这个指针, 应该主动拷贝数据
- C 风格字符串本质上依赖空字符结尾
    - `std::string_view` 不保证以空字符结尾
    - 需要传给 C API 时不要直接假设 `string_view.data()` 可当 C 字符串使用
- `<cctype>` 中的字符分类函数通常接收 `int`
    - 对可能为负的 `char` 直接调用可能出问题
    - 更稳妥的写法是先转换为 `unsigned char`

```C++
bool is_space(char ch) {
    return std::isspace(static_cast<unsigned char>(ch));
}
```

## 容器与迭代器

- `end()` 指向尾后位置, 是哨兵而不是有效元素
- `begin() == end()` 表示范围为空
- 对 `const` 容器调用 `begin()` / `end()` 会得到不可修改元素的迭代器
    - 显式需要只读遍历时可以使用 `cbegin()` / `cend()`
- 可能改变 `std::vector` 容量的操作会使迭代器, 指针和引用失效
    - 典型操作包括扩容时的 `push_back`, `emplace_back`, `insert`
    - `erase` 也会使被删除位置及其之后的迭代器失效
- 范围 `for` 遍历容器时, 不要在循环体内改变该容器的结构
    - 改变大小可能使内部迭代器失效
    - 需要删除元素时优先使用标准算法或 `erase` 返回的新迭代器
- 迭代器距离类型是有符号类型
    - 指针差值对应 `std::ptrdiff_t`
    - 容器大小通常对应无符号的 `std::size_t`

## 数组与指针

- C 风格数组在多数表达式中会退化为首元素指针
    - `auto p = arr` 推导为指针
    - `decltype(arr)` 保留数组类型
    - 引用形参可以保留数组长度信息
- `std::begin(arr)` 和 `std::end(arr)` 可以用于 C 风格数组
    - 新代码中优先考虑 `std::array`, `std::span` 或容器
- 多维数组使用范围 `for` 时, 外层变量应使用引用
    - 否则内层数组会退化为指针

```C++
for (auto& row : matrix) {
    for (auto value : row) {
        // ...
    }
}
```

## 函数接口

- 数组形参会退化为指针
    - `void f(int a[])` 与 `void f(int* a)` 本质接近
    - 如果需要长度, 应显式传入长度, 使用首尾迭代器, 或使用 `std::span`
- `std::initializer_list<T>` 适合表示同类型的轻量可变参数
    - 元素是只读的
    - 拷贝 `initializer_list` 通常只是复制视图, 不复制底层元素
    - 不要让它逃逸出初始化列表底层数组的生命周期
- 默认实参属于调用点
    - 默认实参通常只写在函数声明处
    - 函数指针类型不包含默认实参信息
- 局部作用域中的同名声明会隐藏外层所有同名重载
    - 不是只隐藏签名相同的那一个
    - 类继承中的同名成员函数也有类似问题, 可用 `using Base::func` 引入基类重载集
- 返回引用或指针时, 不要返回局部对象
    - 返回引用的函数调用表达式可以作为左值

## 调试与诊断

- `assert(expr)` 在调试构建中检查表达式
    - 失败时通常打印诊断信息并终止程序
    - 定义 `NDEBUG` 会禁用 `assert`
- 常用预定义信息
    - `__FILE__`: 当前文件名
    - `__LINE__`: 当前行号
    - `__DATE__`: 编译日期
    - `__TIME__`: 编译时间
    - `__func__`: 当前函数名
- 这些宏适合做轻量调试信息
    - 正式日志系统中更推荐使用结构化日志和 `std::source_location`

## 类语义

- 非静态成员函数有一个隐式对象参数
    - 成员函数末尾的 `const` 表示不能通过该隐式对象参数修改对象
    - `const` 对象只能调用 `const` 成员函数
    - `static` 成员函数没有隐式对象参数, 因而不能使用 `this`
- 构造函数初始化列表是真正的初始化
    - 构造函数体内赋值已经是先初始化再赋值
    - `const` 成员, 引用成员, 无默认构造函数的成员必须在初始化列表中初始化
    - 成员初始化顺序由成员声明顺序决定, 不是由初始化列表顺序决定
- 注意最烦人的解析

```C++
MyClass obj;   // 定义对象
MyClass obj(); // 声明函数
```

- 能用一个实参调用的构造函数会参与隐式转换
    - 通常应使用 `explicit` 抑制意外转换
    - `explicit` 也会影响拷贝初始化和拷贝列表初始化
- 友元声明授予访问权限, 但它不是成员
    - 在类内首次声明的友元函数可能不会被普通名字查找直接找到
    - 需要正常调用时, 最好在命名空间作用域也给出函数声明
- 前向声明得到的是不完全类型
    - 可以声明指针或引用
    - 不能定义该类型的普通数据成员
    - 需要访问成员或按值使用对象时必须看到完整定义
- 类外定义成员函数时, 返回类型如果依赖类内类型名, 也需要类名限定

```C++
class Screen {
public:
    using pos = std::string::size_type;
    pos size() const;
};

Screen::pos Screen::size() const {
    return 0;
}
```

- 静态数据成员属于类作用域中的对象, 不是某个实例的一部分
    - C++17 起优先考虑 `inline static` 在类内完成定义
    - 非 `inline` 静态数据成员如果被 ODR-use, 通常仍需要类外定义
    - 静态成员函数不能使用 `this`

## 可不再迁移的旧表述

- `constexpr` 函数只能有一条 `return` 是 C++11 时代限制, 现在不应按这个规则记
- 聚合类型规则已经多次演化, 不要照搬旧书对聚合的条件描述
- 静态成员的类外定义规则受 `inline static` 影响, 不要只记旧写法
- 新代码里不必围绕 C 风格数组建立主知识结构, 应优先转向 `std::array`, `std::span`, 容器和 ranges
