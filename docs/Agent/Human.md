### superpowers skills

- 入口 -> 定义基本约束 + 流程
- 头脑风暴 -> 架构
    - 目标, 约束, 成功标准
    - 通过提问一次澄清一个细节
- 计划 -> 零上下文工程师可执行的任务分解
    - 不要重复
    - 不做多余的工作
    - TDD
- TDD
    - 红: 失败的测试用例
    - 绿: 通过测试用例
    - 重构: 优化代码, 不改变功能
- 执行
    - 自主设定 checkpoint, 要求人类审查
- ReView
    - 左移: 尽可能早地发现问题
    - 细粒度: 每个最小化 Task 都需要 Review
    - 必须提供完整上下文
    - SOLID 原则
- 收到 ReView 反馈后
    - 不能因为配合而全盘接受 ReView 结果 (例如过度设计 / 删除向后兼容)
    - 先验证后执行
    - 敢于反驳
    - 与 ReView 人员完全澄清, 而不是部分接受
- DeBug
    - 根因调查 -> 模式分析 -> 假设验证 -> 修复
    - 三次失败直接放弃, 质疑架构问题
