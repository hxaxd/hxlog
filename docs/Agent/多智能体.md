# 多智能体

## 记忆

- 多智能体的记忆必须支持记录写入者

## 拓扑

### 顺序交接

- 本质是一种压缩方式

### 并行合并

- 增强鲁棒性, 但需要解决冲突
- 可从多角度探索问题

### Teams

- 有主从, 但是从智能体之间也可以有交互 (读写对方邮箱文件), 以协同完成任务
- 也可以使用黑板认领任务 (异步读取同一个黑板)
- 问题高度正交时有效
- 每个成员都需要有生命周期
    - Leader 管理 team config
    - 关机必须经过: 请求 -> 拒绝 / 同意并首尾 -> 确认关机 -> 通知团队并更新 team config
    - 所有消息传递与工作调度都依赖邮箱 + 轮询实现

### Subagent

- 只能主从交流, 本质是声明式的工具调用
