## 编译原理

### 引论

- 编译器, 解释器, Java 语言处理器等常识

#### 编译器的结构

- 源代码 -> 前端 (分析) -> IR (机器无关的中间表示) -> 优化器 -> IR -> 后端 (综合) -> 目标代码
- 前端
    - 原始字符流
        - 预处理器 (处理注释, 宏替换, 条件编译等)
        - `a = b * c + 1;`
    - 词法分析 -> 符号 + 符号表
        - 识别词素, 转化为 Token `<token-name, attribute-value>` (类型 + 符号表中编号, 当然有些类型没有编号)
        - `<ID, 1>` `=` `<ID, 2>` `<*>` `<ID, 3>` `<+>` `<num, 4>`
        - `{1:a, 2:b, 3:c, 4:60}`
    - 语法分析 -> 语法树
        - 变成表达式树
    - 语义分析 -> 语法树
        - 变成
    - 中间代码生成 -> 中间代码
    - 优化器
        - 原始中间代码
        - 优化中间代码 -> 优化后的中间代码
- 后端
    - 优化后的中间代码
    - 目标代码生成 -> 目标代码
    - 目标代码优化 -> 优化后的目标代码
