## 项目经历

### starfish - 轻量级实时推理引擎

**组件一：面向高并发场景的 ai 应用聚合平台**

**项目描述**：

**面向高并发场景的 ai 应用聚合平台**，整合推荐、笔记等多个 ai 应用，支持高并发请求，提供统一的推理 api 接口。

**主要工作**：

* **设计统一 API 网关层**，基于 JWT 实现多租户身份认证与接口标准化，整合异构 AI 应用协议，**屏蔽底层服务差异**；
* **实现高并发请求处理架构**，采用协程池 + 异步 I/O 模型，单节点支持万级并发连接，结合连接复用技术显著**降低上下文切换开销**；
* **设计智能路由与自适应负载均衡**，基于实时负载、GPU 利用率与响应延迟动态调度，**避免热点节点过载**，提升整体资源利用率；
* **设计分布式限流与熔断降级机制**，采用**滑动窗口 + 令牌桶算法**控制流量，结合健康检查与快速失败策略，防止流量洪峰引发雪崩；
* **引入多级缓存架构**，组合**进程内缓存 + 分布式缓存层**，针对热点推理结果与特征数据实现毫秒级响应，显著降低重复计算开销；

---

**组件二：自研分布式特征存储+推理引擎**

**项目描述**：

**自研分布式特征存储+推理引擎**，拥有完整分布式 KV 存储能力，通过将推理逻辑嵌入 KV 存储引擎实现超低延迟场景下的即时推理。

**主要工作**：

* **设计并实现基于跳表的内存存储引擎**，针对 TTL 和区间查询的场景扩展了**有序遍历能力**；
* **采用一致性哈希 + 虚拟节点实现数据分片**，支持首次请求**根据延迟就近选择节点**，后续通过**会话亲和性**路由至同一节点，高负载时支持柔性路由降级，兼顾负载均衡与低延迟需求，并兼容传统分布式 KV 存储的主从复制能力；
* **设计 RDB 快照 + AOF 日志混合持久化方案**，兼顾快速恢复与数据安全性，系统崩溃后数据丢失窗口控制在亚秒级；
* **优化特征存储结构**，采用**稀疏特征压缩与向量化编码**，加速批量特征读取与序列化性能；
* **设计嵌入式推理引擎**，将模型推理下沉至存储层，实现**特征读取**与推理的**零拷贝流水线**，大幅降低端到端延迟；
* **实现动态批处理调度器**，自适应**聚合并发请求进行批量推理**，在满足延迟 SLA 前提下最大化计算资源利用率；
* **引入读写锁与对象内存池**，有效减少锁竞争与频繁堆分配开销，显著提升多线程并发性能。
